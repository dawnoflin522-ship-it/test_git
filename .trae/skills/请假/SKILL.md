# 员工请假申请流程

本skill用于处理员工的请假申请，通过与请假规则比对，判断申请是否符合要求。

## 触发条件

当用户有以下表述时触发：

- "我要请假"
- "申请休假"
- "请年假/病假/事假"
- "想休婚假/产假"
- "查询假期余额"
- "请假几天"
- 任何与请假、休假相关的请求

## 请假信息收集流程

### 第一步：收集必要信息

需要向用户收集以下6项信息，如果用户未提供某一项，必须追问：

1. **所属部门** - 用户所在的部门名称
2. **职位** - 用户的职位名称
3. **请假类型** - 年假、病假、事假、婚假、产假、陪产假、丧假、调休等
4. **请假天数** - 具体请假天数
5. **请假日期范围** - 从哪一天开始到哪一天结束（格式：YYYY-MM-DD 至 YYYY-MM-DD）
6. **请假理由** - 请假的具体原因

**追问策略**：

- 每次只追问缺失的信息，不要一次性追问所有
- 使用礼貌、友好的语气
- 示例："请提供您的所属部门和职位，以便我核对请假权限。"

### 第二步：规则比对

当6项信息全部收集完成后：

1. 读取 references中的[请假规则.md](references/请假规则.md)
2. 根据用户的部门和职位，查找对应的请假权限
3. 根据请假类型，核对以下规则：
   - 该假期类型是否需要提供证明材料
   - 请假天数是否在允许范围内
   - 该职位是否有权限申请此类型假期
   - 是否需要提前申请（如年假需提前3天）
   - 是否存在冲突（如试用期内不可请年假）

**比对结果**：

- 如果符合规则要求 → 返回请假成功回复
- 如果不符合规则要求 → 返回请假失败回复，说明具体原因

## 回复格式

### 请假成功回复模板

```

【申请信息】
- 所属部门：XXX
- 职位：XXX
- 请假类型：XXX
- 请假天数：X天
- 请假日期：XXXX年XX月XX日 至 XXXX年XX月XX日
- 请假理由：XXX

【审核结果】
✓ 符合请假条件
✓ 请假天数在允许范围内
✓ 申请时间符合要求

您的请假申请已通过审核，请按公司流程提交相关证明材料（如需）。
```

```

【申请信息】
- 所属部门：XXX

- 请假类型：XXX
- 请假天数：X天
- 请假日期：XXXX年XX月XX日 至 XXXX年XX月XX日

```

## 各假期类型说明

### 年假

- 根据工作年限确定天数
- 需提前3天申请
- 试用期员工不可申请

### 病假

- 需提供医院证明（1天以上）
- 按实际病情批准
- 病假期间发放基本工资的一定比例

### 事假

- 每年累计不超过20天
- 需提前1天申请
- 事假期间无薪

### 婚假

- 法定婚假3天
- 晚婚额外奖励（根据地方政策）
- 需提供结婚证复印件
- 需在结婚登记后1年内使用

### 产假

- 女性员工生育享受
- 按国家规定执行（通常98天+地方奖励）
- 需提供医院证明

### 陪产假

- 男性员工配偶生育享受
- 通常7-15天（根据地方政策）
- 需提供出生证明

### 丧假

- 直系亲属去世可享受
- 通常3天
- 需提供相关证明

### 调休

- 加班后可申请调休
- 需有加班记录
- 调休有效期通常为3个月

## 异常处理

如果在处理过程中遇到任何异常：

- 告知用户："请假服务异常，请稍后再试。"
- 不要暴露具体的技术错误信息

## 注意事项

1. **信息完整性**：必须确保6项信息全部收集完成才能进入规则比对
2. **规则准确性**：严格按照reference文件中的规则执行，不得主观判断
3. **隐私保护**：处理过程中涉及的员工个人信息不得记录或泄露
4. **专业态度**：保持专业、友好的沟通方式
5. **证明材料**：提醒用户在需要时提交相关证明材料
